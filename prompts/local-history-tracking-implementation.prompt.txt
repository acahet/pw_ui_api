# Playwright Local Test History Tracking Implementation

## Overview
Implement a local test history tracking system for Playwright that automatically tracks test runs, preserves failure artifacts (traces, screenshots), and provides a visual HTML dashboard - all without external dependencies.

## Requirements

### Core Features
1. **Automatic history tracking** for all test runs
2. **7-day automatic cleanup** of old test runs and artifacts
3. **Failure artifact preservation** (traces and screenshots)
4. **HTML dashboard** to visualize test trends
5. **Exclude setup project** from tracking
6. **Preserve existing Playwright config** (trace, screenshot settings)

### Technical Stack
- TypeScript for reporter implementation
- Playwright Reporter API
- Pure HTML/CSS/JavaScript for dashboard (no build tools)
- Node.js fs/promises for file operations

- **Artifact copying**: Scan project outputDir for traces/screenshots

## Data Structure

### History Index Format

```typescript
interface TestHistoryEntry {
  runId: string;                    // ISO timestamp with special chars replaced
  timestamp: string;                // ISO 8601 timestamp
  duration: number;                 // Run duration in milliseconds
  totalTests: number;               // Total tests executed
  passed: number;                   // Passed tests
  failed: number;                   // Failed tests
  skipped: number;                  // Skipped tests
  flaky: number;                    // Flaky tests (passed after retry)
  failedTests: Array<{
    title: string;                  // Test title
    file: string;                   // Relative file path
    project: string;                // Project name
    duration: number;               // Test duration in ms
    error?: string;                 // Error message
    artifacts: {
      trace?: string;               // Relative path to trace file
      screenshot?: string;          // Relative path to screenshot
    };
  }>;
}

interface HistoryIndex {
  runs: TestHistoryEntry[];
}
```

### Customization Options

**Change retention period**:
```typescript
// In history-reporter.ts
const cutoffDate = Date.now() - 14 * 24 * 60 * 60 * 1000; // 14 days
```

**Change max runs stored**:
```typescript
// In history-reporter.ts

```

**Modify dashboard styling**:

## Common Issues & Solutions

### Dashboard shows "No Test History"
- **Solution**: Run tests first to generate history data

### Artifacts not copying
- **Check**: Project outputDir exists and contains artifacts
- **Check**: Failed tests actually produced traces/screenshots
- **Check**: File permissions allow copying

### Cleanup not working
- **Check**: Dates in history-index.json are valid ISO strings
- **Check**: File system permissions
- **Check**: Console for error messages

### Dashboard not loading
- **Solution**: Serve via HTTP server (not file:// protocol)
- **Use**: `npm run report:history` or `http-server`

## Benefits Over playwright-smart-reporter

1. **Local-only** - No external dependencies or services
2. **Full control** - Modify any aspect of tracking
3. **Artifact preservation** - Automatically copies and preserves failure artifacts
4. **Automatic cleanup** - 7-day retention with no manual intervention
5. **Simple setup** - No npm package installation required
6. **Customizable** - Full access to source code

## Performance Considerations

- **Artifact copying**: Only failed tests have artifacts copied
- **History limit**: Maximum 30 runs stored by default
- **Cleanup timing**: Runs after each test execution
- **File I/O**: Asynchronous operations prevent blocking

## Security Notes

- All files stored locally
- No external API calls
- No sensitive data in artifacts (traces may contain tokens - be careful)
- Dashboard runs entirely client-side (no server processing)
